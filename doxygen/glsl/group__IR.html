enum ir_node_type

Class tags.

Each concrete class derived from ir_instruction has a value in this enumerant. The value for the type is stored in ir_instruction::ir_type by the constructor. While using type tags is not very C++, it is extremely convenient. For example, during debugging you can simply inspect ir_instruction::ir_type to find out the actual type of the object.

In addition, it is possible to use a switch-statement based on ir_instruction::ir_type to select different behavior for different object types. For functions that have only slight differences for several object types, this allows writing very straightforward, readable code.

Enumerator
ir_type_unset
