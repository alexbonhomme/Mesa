GLboolean _mesa_check_conditional_render ( struct gl_context *  ctx)

This function is called by software rendering commands (all point, line triangle drawing, glClear, glDrawPixels, glCopyPixels, and glBitmap, glBlitFramebuffer) to determine if subsequent drawing commands should be executed or discarded depending on the current conditional rendering state.

Ideally, this check would be implemented by the GPU when doing hardware rendering. XXX should this function be called via a new driver hook?

GL_TRUE if we should render, GL_FALSE if we should discard
